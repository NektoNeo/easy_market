
# Product — “Упаковка товара для маркетплейса за 3 минуты”

## Цель
Дать продавцу (селлеру/контент‑менеджеру) быстрый, **структурированный** и **безопасный** текстовый комплект для карточки товара (WB/Ozon/и др.) за ~3 минуты.

## Основные пользователи (персоны)
1. **Селлер‑одиночка** — сам ведёт ассортимент, хочет экономить время на текстах.
2. **Контент‑менеджер** — делает много карточек, важны шаблоны и скорость.
3. **Агентство/менеджер кабинета** — ведёт нескольких клиентов, нужны профили бренда и история.

## Что входит в MVP
- Wizard 3 шага → Generate → Results (табы) → History.
- Шаблоны (seed templates) для популярных категорий.
- Профили бренда (на уровне приложения: тон, запрещённые слова).
- Мок‑провайдер LLM по умолчанию (детерминированный) + абстракция под Yandex/GigaChat.
- Контроль рисков: `risk_flags`, `compliance_notes`, `missing_info_questions`.

## Что НЕ входит в MVP (non-goals)
- Автозагрузка фото/скан упаковки (CV).
- Полная интеграция платежей/подписок (есть архитектурные заглушки).
- Синхронизация истории между устройствами.
- Полноценная CRM интеграция.

## Метрики успеха (MVP)
- Time-to-first-result < 3 минут.
- Конверсия Wizard → Generate ≥ 70% (в тестовой аудитории).
- Повторные генерации (history opens) ≥ 30%.
- Жалобы на “выдуманные факты” → 0 (или стремиться к нулю).

## Принципы безопасности и комплаенса
- Нельзя утверждать то, чего нет во входных данных.
- Если данных не хватает — задавать уточняющие вопросы (`missing_info_questions`).
- Любые рискованные формулировки маркировать `risk_flags`.
